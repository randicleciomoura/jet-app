# Frontend Source

Pasta raiz do código do frontend.

## Padrão de pastas

- `app/` para composição inicial e providers.
- `components/` para componentes reutilizáveis.
- `features/` para casos de uso por domínio.
- `schemas/` para contratos de interface.
- `stores/` para estado global.
- `services/` para acesso a APIs e persistência.
- `types/` para tipos locais.

Os diretórios reais serão criados conforme a implementação avançar.
